IMMICH EXTERNAL LIBRARY RESEARCH - QUICK FINDINGS
==================================================

QUESTION 1: Does Immich have a native external library feature?
ANSWER: YES - External Libraries is a fully supported, first-class feature
- Official docs: https://immich.app/docs/features/libraries
- Web UI: Administration > External Libraries
- Create/edit/delete via UI
- Per-user ownership

QUESTION 2: Official supported methods for external storage?
ANSWER: Three methods
1. Docker Volume Mounts (for Docker deployments)
   Format: immich-server volumes: [ /host/path:/container/path:ro ]
   
2. Filesystem-Level Mounts (for NixOS/systemd)
   Used in current nixos-dotfiles configuration
   
3. Network Mounts (NAS/SMB/NFS)
   Supported but auto-watching doesn't work on networks

QUESTION 3: Is bind mounting recommended or is there a better way?
ANSWER: For NixOS - bind mounting IS correct and recommended
- Your current approach is sound
- Matches Immich's official troubleshooting steps
- Clean integration with NixOS philosophy
- Alternative: Use External Libraries UI for more flexibility

QUESTION 4: How does Immich handle multiple storage locations?
ANSWER: Three approaches
1. Multiple import paths in single library
   - One library references /path1, /path2, /path3
   - Deduplication across all paths
   
2. Storage template engine (for uploaded files)
   - Organize by pattern: Year/Year-Month-Day/Filename
   - Customizable via UI
   
3. Multiple external libraries
   - Each library owned by single user
   - Independent scanning schedules

QUESTION 5: Required permissions/ownership for external libraries?
ANSWER: Standard NixOS/Linux permissions
- Immich runs as user: immich
- Minimum: read permission (rx) for directories
- For metadata writes: rw permissions needed
- Current config includes: users.users.immich.extraGroups = [ "users" ]
- Read-only: Add "ro" option to mount for archives
- Best practice: Use :ro in Docker, "ro" in NixOS mounts for archives
